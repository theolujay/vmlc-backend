# Export and Import Exam Commands

This document explains how to use the Django management commands for exporting and importing exams in the VMLC backend.

## Exporting an Exam

The `export_exam` command allows you to export an existing exam, its questions, and its stage-linking information into a JSON file.

### Usage
```bash
python manage.py export_exam <exam_id> [--output <file_path>]
```

### Arguments
- `exam_id` (Required): The UUID of the exam you want to export.
- `--output`, `-o` (Optional): The file path where the JSON data should be saved. If omitted, the JSON will be printed to `stdout`.

### Example
```bash
python manage.py export_exam 42fca4df-8845-4a8f-9635-f52986af4ac8 --output exam_export.json
```

## Importing an Exam

The `import_exam` command allows you to create a new exam and its questions from a JSON file generated by `export_exam`.

### Usage
```bash
python manage.py import_exam <file_path> [--staff-email <email>] [--competition-id <id>]
```

### Arguments
- `file_path` (Required): The path to the JSON file to import.
- `--staff-email` (Optional): The email of the staff user who should be marked as the creator of the exam and questions. If not provided, it defaults to the first staff member found in the system.
- `--competition-id` (Optional): The ID of the competition to link the exam to. If provided, and the JSON contains `stage_exam` info, the command will attempt to link the new exam to the corresponding stage and round in that competition.

### Example
```bash
python manage.py import_exam exam_export.json --staff-email admin@vmlc.com --competition-id 1
```

## Technical Details

### Image Handling
- **Export**: Question images are exported as their file names (e.g., `"question_images/sample.jpg"`).
- **Import**: The command attempts to set the `image` field of the question to the filename provided in the JSON. Note that the actual image files must already exist in the media storage for them to be accessible in the application.

### Stage Linking
If an exam is linked to a `StageExam` (e.g., a specific round in a competition), that metadata is included in the export. During import, if a `--competition-id` is provided, the command will look for a matching `Stage` (Screening, League, or Final) and `Round` in that competition to establish the link.

### Atomicity
The import process is wrapped in a database transaction. If any part of the import fails (e.g., invalid JSON or missing required fields), the entire operation is rolled back, and no records are created.
