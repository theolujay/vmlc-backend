# User Roles, Permissions, and API Access

This table provides a detailed breakdown of each user role, its key abilities on the platform, and the primary API endpoints it has access to.

### Candidate User Type

| Role            | Key Abilities (What they can do)                                                                                                                                  | Accessible API Endpoints                                                                                                                                                                                                                                                                           |
| --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`screening`** | ‚ÄĘ View their personal dashboard.<br>‚ÄĘ Take screening-round exams.<br>‚ÄĘ View their own profile and verification status.<br>‚ÄĘ View screening leaderboard snapshots. | ‚ÄĘ `GET /dashboard/candidate/`<br>‚ÄĘ `GET /candidates/me/`<br>‚ÄĘ `GET /user/verification/status/`<br>‚ÄĘ `POST /user/verification/upload/`<br>‚ÄĘ `GET /exams/{id}/take-exam/` (where `stage` is `screening`)<br>‚ÄĘ `POST /exams/{id}/submit-exam-answers/`<br>‚ÄĘ `GET /leaderboard/` (for screening exams) |
| **`league`**    | ‚ÄĘ All `screening` abilities.<br>‚ÄĘ Take league-round exams.<br>‚ÄĘ View the competition leaderboard snapshots or a specific exam leaderboard.                        | ‚ÄĘ All `screening` endpoints.<br>‚ÄĘ `GET /leaderboard/`<br>‚ÄĘ `GET /exams/{id}/take-exam/` (where `stage` is `league`)                                                                                                                                                                                |
| **`final`**     | ‚ÄĘ All `league` abilities.<br>‚ÄĘ Access to (offline) final-stage exams.                                                                                             | ‚ÄĘ All `league` endpoints.<br>‚ÄĘ `GET /exams/{id}/take-exam/` (where `stage` is `final`)                                                                                                                                                                                                             |
| **`winner`**    | ‚ÄĘ Ceremonial role with all candidate permissions. Registered winner of the final stage.                                                                           | ‚ÄĘ All `final` endpoints.                                                                                                                                                                                                                                                                           |

### Staff User Type

_(Permissions are hierarchical; higher roles inherit permissions from lower roles)_

| Role             | Key Abilities (What they can do)                                                                                                                                                                                        | Newly Accessible API Endpoints (in addition to lower roles)                                                                                                                                                                                                                                         |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **`volunteer`**  | ‚ÄĘ View their own profile.<br>‚ÄĘ Submit their own documents for verification.                                                                                                                                             | ‚ÄĘ `GET /staff/me/`<br>‚ÄĘ `GET /user/verification/status/`<br>‚ÄĘ `POST/PATCH /user/verification/upload/`                                                                                                                                                                                               |
| **`admin`**      | ‚ÄĘ View details for any candidate.<br>‚ÄĘ Change roles for candidates.<br>‚ÄĘ Full management (CRUD) of exams.<br>‚ÄĘ Manually submit scores.<br>‚ÄĘ Publish leaderboard for a specific exam.                                    | ‚ÄĘ `GET /candidates/{id}/`<br>‚ÄĘ `GET /candidates/{id}/scores/`<br>‚ÄĘ `GET /candidates/{id}/exam-history/`<br>‚ÄĘ `PUT /candidates/{id}/roles/assign/`<br>‚ÄĘ `GET/POST /exams/`<br>‚ÄĘ `GET/PUT/PATCH/DELETE /exams/{id}/`<br>‚ÄĘ `PUT /exams/{id}/submit-exam-score/`<br>‚ÄĘ `POST /leaderboard/publish/`      |
| **`manager`**    | ‚ÄĘ View details for any staff member.<br>‚ÄĘ Change roles for staff (except `manager` or `superadmin`).<br>‚ÄĘ Manage user verifications for candidates and staff members (approve/reject).<br>‚ÄĘ Create and view broadcasts. | ‚ÄĘ `GET /staff/{id}/`<br>‚ÄĘ `PUT /staff/{id}/roles/assign/`<br>‚ÄĘ `GET /user/verification/list/`<br>‚ÄĘ `POST /user/verification/action/{id}/`<br>‚ÄĘ `GET /user/verification/documents/{type}/{id}/`<br>‚ÄĘ `GET/POST /broadcasts/`<br>‚ÄĘ `GET /broadcasts/{id}/`<br>‚ÄĘ `GET/PATCH /account-management/{id}/` |
| **`superadmin`** | ‚ÄĘ Can assign any staff role (except `superadmin`).<br>‚ÄĘ Has full platform control inheriting all permissions.                                                                                                           | _(Inherits all `manager` endpoints with zero restrictions)_                                                                                                                                                                                                                                         |
| **`sponsor`**    | ‚ÄĘ A vanity role with no specific permissions.                                                                                                                                                                           | _(No specific endpoints)_                                                                                                                                                                                                                                                                           |

**NOTE**: Users must already have their emails verified and be user-verified to perform actions beyond `get-started`.

#### Role Promotion

- **Candidates**: `screening` ‚Üí `league` ‚Üí `final` ‚Üí `winner` (promotion is managed by staff with `admin` role or higher)
- **Staff**: Roles are assigned by a `manager` or `superadmin`.
